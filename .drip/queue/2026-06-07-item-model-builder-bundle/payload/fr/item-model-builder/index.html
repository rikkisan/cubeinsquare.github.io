Pourquoi l’utiliser

  • Ne réécrivez plus la structure model/select/range_dispatch à chaque fois.
  • Gardez le dossier items lisible pour toute l’équipe.
  • Testez une definition avant de lancer une migration plus large du pack.

Avant l’export

  • Vérifiez que les chemins de modèles existent déjà dans le resource pack.
  • Gardez un fallback sûr pour éviter les erreurs visuelles si une branche manque.
  • Si la logique dépend de custom_model_data, vérifiez l’index et la stratégie de slot côté commande.
