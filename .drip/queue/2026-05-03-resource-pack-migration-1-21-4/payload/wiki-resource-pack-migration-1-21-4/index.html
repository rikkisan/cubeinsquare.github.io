Minecraft 1.21.4 Migration Guide: Moving Resource Packs to the Component Era

Minecraft 1.21.4 does not only change command spelling. It changes how your pack workflow thinks about item identity, potion payloads, custom model references, and every hand-written give command you still keep in old notes.

Open resource-pack generator

What actually changed

The painful part of the migration is not the ZIP pack itself. Textures, models, and blockstates still live in the same folders. The real shift is item data: commands moved away from old NBT-style item tags and into explicit components.

The first thing to audit

  • Saved /give commands in docs or command blocks
  • Villager trade commands that embed item payloads
  • Potion commands and NPC rewards
  • Any CustomModelData items used in quests, shops, or scripts

Old and new mindset

/give @p minecraft:paper{CustomModelData:12,display:{Name:'{"text":"Pass"}'}} 1
/give @p minecraft:paper[minecraft:custom_model_data=12,minecraft:custom_name='[{"text":"Pass"}]'] 1

The new syntax is flatter, easier to compose, and better for long-term maintenance once you stop thinking in nested NBT blobs.

Migration checklist

  1. List every old command your server still uses.
  2. Convert visible fields like names and lore to dedicated components.
  3. Move hidden identity checks into minecraft:custom_data.
  4. Re-test villager trades, quest rewards, and potion items on a staging world.
  5. Update wiki notes and admin cheat sheets so staff stop pasting old syntax.

Resource-pack files vs command files

Your resource-pack ZIP often survives the version jump almost untouched. The parts that fail first are admin commands, loot distribution, shops, NPCs, and story items. Treat migration as a command-and-data audit, not only as a texture-pack update.

Where the generator helps

The pack generator already separates target version choices. That means you can keep the art pipeline stable while changing the command syntax that staff and builders copy from the tools.

What to retest after the move

  • Every item that uses custom_model_data
  • Every potion with custom contents
  • Every villager that validates a special item
  • Every admin macro or staff note with a saved command
