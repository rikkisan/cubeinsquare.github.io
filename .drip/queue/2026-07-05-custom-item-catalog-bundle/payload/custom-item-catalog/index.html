Minecraft Custom Item Catalog

This is the planning layer between "we have a few cool items" and "we now have thirty identifiers and nobody remembers which paper is which". Keep a local registry, search it, export it, and stop guessing.

Read guide Open builder

Catalog entry

Stored in this browser

Use this as a local working registry while you sketch IDs, hooks, and notes. Export JSON or CSV when you want to hand the plan to the rest of the team.

Why keep a catalog?

  • Duplicate hidden IDs are easier to catch before they become quest bugs.
  • Base item, display name, and visual hook stay readable in one place.
  • The export gives the rest of the team something more useful than screenshots.

Good first fields

  • Start with an internal ID, hidden ID, and base item.
  • Then add either item_model or CustomModelData depending on the version you ship.
  • Use notes for quest rules, NPC ownership, or pack-side reminders that will matter again later.
