Как сделать обратный отсчёт bossbar в Minecraft через командные блоки

Bossbar может показывать таймер, но сама шкала не отсчитывает время. Команда настройки создаёт bar, а значение должен обновлять другой слой: повторяющийся командный блок, цепочка команд или datapack-функция.

Открыть Bossbar BuilderЧитать гайд по bossbar

Сначала создайте bossbar

Добавьте именованную bossbar, задайте цвет, стиль, максимум, текущее значение и игроков. Builder делает эту настройку аккуратно. Для таймера на 60 секунд поставьте max 60 и value 60.

Используйте scoreboard для состояния

Scoreboard objective хранит число, которое Minecraft будет уменьшать. Потом это число можно копировать в значение bossbar, чтобы видимая шкала была связана с реальным состоянием сервера.

Обновляйте значение повтором

Repeating command block или scheduled datapack function должны уменьшать время и обновлять bossbar value. Одна статичная /bossbar-команда не делает живой отсчёт.

Завершайте чисто

Когда таймер дошёл до нуля, спрячьте или удалите bossbar, запустите событие и остановите обновление. Старые висящие шкалы путают следующие сцены.

Шаг

ШагПример командыЗачем
Создать objective/scoreboard objectives add event_timer dummyХранит оставшееся время.
Задать время/scoreboard players set #timer event_timer 60Старт с 60 секунд.
Показать шкалу/bossbar set cube:event value 60Первое видимое значение.
Уменьшать/scoreboard players remove #timer event_timer 1Шаг повторяющегося обновления.
Скопировать значение/execute store result bossbar cube:event value run scoreboard players get #timer event_timerСинхронизирует scoreboard и bossbar.

FAQ

Можно ли сделать живой таймер одной командой?

Нет. Создать шкалу можно одной командой, но отсчёт требует повторяющихся обновлений.

Лучше секунды или тики?

Секунды проще для command blocks. Тики удобнее, если событием уже управляет datapack.

Что делать на нуле?

Запустить итоговую команду, затем скрыть или удалить bar.
