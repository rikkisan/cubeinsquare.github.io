Créer un compte à rebours Minecraft avec bossbar et command blocks

Une bossbar peut afficher un minuteur, mais elle ne compte pas toute seule. La commande crée la barre, puis une couche de mise à jour doit modifier sa valeur : command block répétitif, chaîne de commandes ou fonction datapack.

Ouvrir Bossbar BuilderLire le guide bossbar

Créez d’abord la bossbar

Ajoutez une bossbar nommée, puis définissez couleur, style, max, valeur et joueurs. Le builder prépare cette base proprement. Pour 60 secondes, mettez max à 60 et value à 60.

Stockez le temps dans un scoreboard

Un objective scoreboard donne à Minecraft un nombre à décrémenter. Ce nombre peut ensuite être copié dans la valeur de la bossbar.

Mettez la valeur à jour en boucle

Un command block répétitif ou une fonction datapack planifiée doit diminuer le temps et mettre à jour la bossbar. Une commande /bossbar statique ne suffit pas.

Terminez proprement

À zéro, cachez ou supprimez la bossbar, lancez l’événement et arrêtez la boucle. Les anciennes barres rendent les scènes suivantes confuses.

Étape

ÉtapeCommande exempleRôle
Créer l’objectif/scoreboard objectives add event_timer dummyStocke le temps restant.
Définir le temps/scoreboard players set #timer event_timer 60Démarre à 60 secondes.
Afficher la barre/bossbar set cube:event value 60Valeur visible initiale.
Décrémenter/scoreboard players remove #timer event_timer 1Étape répétée.
Copier la valeur/execute store result bossbar cube:event value run scoreboard players get #timer event_timerSynchronise scoreboard et bossbar.

FAQ

Une commande suffit-elle pour un minuteur vivant ?

Non. La barre se crée avec une commande, mais le compte à rebours demande des mises à jour répétées.

Secondes ou ticks ?

Les secondes sont plus simples en command blocks. Les ticks conviennent mieux aux datapacks.

Que faire à zéro ?

Lancez l’action finale puis cachez ou supprimez la barre.
