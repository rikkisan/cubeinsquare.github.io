RUENFRDE

Minecraft Custom Item Builder

Erstellen Sie fertige /give-Befehle für RP-Requisiten, Quest-Belohnungen und Gildenpässe. Dieser Builder trennt die Versionen, sodass Sie bei älterer Syntax kompakt bleiben oder item_model, custom_data und reichhaltige custom_model_data-Slots für 1.21.4+ nutzen können.

Pack-Generator öffnen CustomModelData-Guide lesen

Item-Setup

Verwenden Sie das Vanilla-Item, das der Spieler erhält (z.B. paper, stick oder book).

Eine Logik-ID für Skripte, Quests und Admin-Prüfungen. Sie bleibt für Spieler verborgen.

Was ändert sich mit der Version?

Verwenden Sie 1.21.4+, wenn Sie item_model, lesbare String-Zustände, Float-, Flag- oder Color-Slots in custom_model_data benötigen.

Älteres Format (bis 1.21.3)

Ältere Formate sind kompakt: ein numerischer CustomModelData-Wert, ein sichtbarer Name, Lore und eine optionale versteckte Item-ID.

1.21.4+: item model und CMD-Slots

Moderne Items trennen Rendering und Logik sauber: item_model verweist auf die Kunst, während custom_model_data Float-, String-, Flag- und Color-Slots zusammen tragen kann.

Verwenden Sie einen vollständigen Pfad wie cubeinsquare:item/path_here. Lassen Sie es leer, wenn Sie nur eine versteckte ID benötigen.

Generierter Befehl

Version: 1.21.4+. Befehlslänge: 0 Zeichen.

Der Builder hält die ältere Syntax absichtlich kompakt. Beim Wechsel zu 1.21.4+ erweitert sie sich in das reichhaltigere Komponentenformat.

Diese Version kann gleichzeitig eine visuelle Modellroute, eine lesbare versteckte ID und reichhaltige custom_model_data-Slots tragen.

Wofür das gut ist

  • RP-Währungen, Genehmigungen, Verträge, Hinweise, Schlüssel, Relikte und Gildenrequisiten
  • Quest-Belohnungen, die eine versteckte Logik-ID und ein separates visuelles Modell benötigen
  • Serverseitige Validierung, bei der eine lesbare Item-Identität wichtiger ist als eine Zufallszahl
  • Schnelles Testen, wenn Sie zuerst den Befehl und dann die Pack-Pipeline wollen
